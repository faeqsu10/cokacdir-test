# Ontology Review Skill

구현된 온톨로지와 코드의 품질을 검증하는 리뷰 스킬.

## 리뷰 체크리스트

### 온톨로지 스키마 리뷰
- [ ] 엔티티/관계 네이밍 일관성 (kebab-case)
- [ ] 필수 속성 누락 여부
- [ ] 추론 규칙의 논리적 정합성
- [ ] 순환 참조 없음
- [ ] 재난 도메인 완전성 (주요 재난 유형 커버)

### 예측 모델 리뷰
- [ ] 데이터 전처리 적절성 (결측치, 이상치 처리)
- [ ] 특성 선택 근거
- [ ] 학습/검증 분리 (교차 검증)
- [ ] 평가 메트릭 적절성
- [ ] 과적합 징후 확인

### LLM 연동 리뷰
- [ ] 프롬프트 품질 (명확성, 컨텍스트 제공)
- [ ] 에러 핸들링 (API 실패, 타임아웃)
- [ ] TypeQL 생성 정확성
- [ ] 토큰 사용 효율성

### 코드 품질 공통
- [ ] 실행 가능한 상태
- [ ] 에러 핸들링 존재
- [ ] 주석/문서화 적절
- [ ] 민감 정보 노출 없음

## 리뷰 프로세스

### 1단계: 자체 리뷰
코드를 직접 읽고 위 체크리스트 기준으로 분석한다.

### 2단계: Codex 독립 리뷰
```bash
codex "Review {file_path}. Focus on: correctness, security, performance, readability. Rate each 1-10 and suggest concrete improvements."
```

### 3단계: 종합 리뷰 문서 작성

```markdown
# 리뷰 결과: {파일/모듈명}

## 점수
| 항목 | 점수 | 비고 |
|------|------|------|
| 정확성 | /10 | |
| 보안 | /10 | |
| 성능 | /10 | |
| 가독성 | /10 | |
| **종합** | **/10** | |

## 주요 이슈
1. [심각도: 높음/중간/낮음] {이슈 설명}
   - **현재:** {문제 코드}
   - **개선안:** {수정 코드}

## Codex 의견
{Codex 리뷰 요약}

## 결론
{통과/조건부 통과/수정 필요}
```

## 증분 리뷰
모듈 완성 직후 바로 리뷰한다 (전체 완성까지 기다리지 않음).
builder가 `src/ontology/schema.tql` 완료 → 즉시 스키마 리뷰
builder가 `src/models/predictor.py` 완료 → 즉시 모델 리뷰
